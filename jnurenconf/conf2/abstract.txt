- 分享者:

- 分享内容:

"LSD 是什么? 是那个著名的致幻剂吗?!" 当然不是.

LSD 即 legibility (可读性), semantics (语义) 和 delegation (委托) 三个名词的简称,
也是在 Web 开发中不可脱离的三个因素. 随着 HTML5 这一名词在近几年中的墙裂爆发, 以及 W3C
理事会对新标准 HTML5.1 的官方推荐, 还有数不清的各种 Web 标准的更新和变革, 作为一个正在学习
Web 开发的小白应该要注意哪些规范和原则呢?

没错! 就是 LSD!

本次的技术沙龙, 将会从可读性, 语义, 委托这三个方面递进地讲解其中的好玩儿的地方, 帮助了解 HTML
标准, 浏览器支持, 甚至是 ES6 等标准中的极佳的实践姿势! 让你开始摆脱对工具, 框架等繁重的依赖!

"怎么不用 Bootstrap 等框架, 快速实现响应式?"

"怎么做出来一个连残疾人都能使用的网页?"

"怎么快速地设计一个 app 的原型?"

"浏览器还能允许我做什么厉害的羞羞的事情吗?"

想知道答案? 到时见分晓!
